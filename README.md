# LegalHelp — юридический AI-помощник с гибридным RAG

**LegalHelp** — это локальный RAG-ассистент для подготовки юридических ответов и процессуальных документов.  
Проект разработан как **open-source альтернатива коммерческим СПС** (КонсультантПлюс, Гарант) с фокусом на **конфиденциальность** (данные не покидают компьютер пользователя, соответствие 152-ФЗ).

---

## 🎯 Цель проекта

Создать интеллектуальную систему, которая:
- Отвечает на юридические вопросы со ссылками на конкретные статьи кодексов.
- Генерирует процессуальные документы (возражения, жалобы, иски) по шаблонам.
- Работает полностью локально, без передачи данных в облако.
- Обеспечивает точность поиска не ниже коммерческих аналогов.

## 🛠️ Технологический стек

| Компонент | Технология |
|-----------|------------|
| **Бэкенд** | FastAPI + ChromaDB (13 500+ чанков законов) |

| **Фронтенд** | React + Vite |
| **OCR** | EasyOCR |
| **Модель эмбеддингов** | `intfloat/multilingual-e5-large` |
| **Reranking** | `BAAI/bge-reranker-v2-m3` |

## 🔍 Архитектура поиска (гибридный RAG)

Система использует **5-этапный пайплайн** для максимальной точности:

1. **Query Expansion** — расширение запроса юридическими синонимами с лемматизацией (pymorphy3).
2. **Семантический поиск** (bi-encoder E5-Large) → топ-30 кандидатов.
3. **BM25 поиск** с лемматизацией → топ-30 кандидатов.
4. **Reciprocal Rank Fusion (RRF)** — объединение результатов обоих поисков.
5. **Reranking** (cross-encoder bge-reranker-v2-m3) → финальный топ-5.

## 📊 Метрики качества (по результатам ML-экспериментов)

| Стратегия | Precision@5 | MRR |
|-----------|-------------|-----|
| Векторный поиск (только эмбеддинги) | 0.62 | 0.58 |
| **Гибридный поиск (BM25 + эмбеддинги + RRF + Reranking)** | **0.87** | **0.84** |

| Конфигурация | Верность ответов (Faithfulness) |
|--------------|--------------------------------|
| Без RAG (голая LLM) | 66% (34% галлюцинаций) |
| RAG (векторный) | 89% |
| **RAG + гибридный поиск** | **92%** |

**Вывод:** гибридный RAG снижает галлюцинации на **25–30%** по сравнению с голой LLM.

## 🗂️ Конвертация данных (Akoma Ntoso + FRBR)

Для импорта правовых актов в систему используется стандарт **Akoma Ntoso** — международный XML-формат для представления юридических документов. В проекте реализован конвертер (`src/converters/akoma_ntoso_converter.py`), который:

- Парсит структуру кодекса (статьи, пункты, подпункты).
- Генерирует XML-файл с метаданными **FRBR** (Functional Requirements for Bibliographic Records) — описывает уровень документа, дату принятия, номер, ссылки и т.д.
- Создаёт структурированные чанки для загрузки в ChromaDB через `structured_chunker.py`.

Это позволяет автоматически загружать кодексы из исходных XML-файлов (например, из репозитория RusLawOD) в едином формате, обеспечивая точный поиск по статьям и пунктам.

## 🚀 Ключевые возможности

- **Ответы на вопросы** с цитированием статей и проверкой источников (CitationVerifier).
- **Генерация документов** по шаблонам (возражения, жалобы, иски).
- **Загрузка файлов** (PDF, изображения) с распознаванием текста (OCR).
- **Полная локальность** — никаких внешних API, данные не покидают ваш компьютер.
- **Интерактивный веб-интерфейс** (React) для удобной работы.
